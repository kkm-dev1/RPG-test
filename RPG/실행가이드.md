# 🎮 RPG 게임 실행 가이드

## 📦 1단계: 패키지 설치

터미널(명령 프롬프트 또는 PowerShell)을 열고 프로젝트 폴더로 이동한 후 다음 명령어를 실행하세요:

```bash
pip install -r requirements.txt
```

또는 개별 설치:

```bash
pip install arcade pygbag
```

## 💻 PC에서 실행하기

### 방법 A: 웹 브라우저에서 실행 (권장)

프로젝트 폴더에서 다음 명령어를 실행:

```bash
pygbag main.py
```

**실행 후:**
- 자동으로 웹 브라우저가 열립니다
- 로컬 서버가 시작됩니다 (기본 주소: `http://localhost:8000`)
- 게임이 브라우저에서 실행됩니다

**게임 조작:**
- 방향키 (↑ ↓ ← →)로 파란색 사각형을 이동시킬 수 있습니다

### 방법 B: 일반 Python 실행 (개발/테스트용)

웹 변환 없이 직접 실행:

```bash
python main.py
```

## 📱 모바일에서 실행하기

### 방법 1: 같은 Wi-Fi 네트워크 사용 (로컬 네트워크)

#### PC에서:

1. **PC에서 pygbag 실행:**
   ```bash
   pygbag main.py
   ```

2. **PC의 IP 주소 확인:**
   
   **Windows:**
   ```bash
   ipconfig
   ```
   - 출력에서 "IPv4 주소" 찾기
   - 예: `192.168.0.100`

   **Mac/Linux:**
   ```bash
   ifconfig
   ```
   또는
   ```bash
   ip addr show
   ```

3. **방화벽 설정 확인:**
   - Windows 방화벽에서 포트 8000 허용 필요할 수 있음
   - 보안 경고가 뜨면 "액세스 허용" 클릭

#### 모바일에서:

1. **모바일 기기를 PC와 같은 Wi-Fi 네트워크에 연결**

2. **모바일 브라우저 열기** (Chrome, Safari 등)

3. **주소창에 입력:**
   ```
   http://PC의IP주소:8000
   ```
   예: `http://192.168.0.100:8000`

4. **게임 실행!**
   - 모바일에서는 가상 키보드를 사용하거나 터치 컨트롤 추가 필요

### 방법 2: 웹 호스팅 서비스 사용 (인터넷 공개)

#### 1단계: 빌드 파일 생성

```bash
pygbag --build main.py
```

이 명령어는 `build/web/` 폴더에 웹 파일을 생성합니다.

#### 2단계: 웹 호스팅 서비스에 업로드

**옵션 A: GitHub Pages (무료)**

1. GitHub 저장소 생성
2. `build/web/` 폴더의 내용을 저장소에 업로드
3. Settings > Pages에서 GitHub Pages 활성화
4. 생성된 URL로 접속

**옵션 B: Netlify (무료, 추천)**

1. [netlify.com](https://www.netlify.com) 가입
2. "Add new site" > "Deploy manually"
3. `build/web/` 폴더를 드래그 앤 드롭
4. 자동으로 URL 생성됨

**옵션 C: Vercel (무료)**

1. [vercel.com](https://vercel.com) 가입
2. 프로젝트 가져오기
3. `build/web/` 폴더 선택
4. 배포 완료

#### 3단계: 모바일에서 접속

생성된 URL을 모바일 브라우저에서 열기

## 🔧 고급 설정

### 다른 포트 사용하기

기본 포트 8000이 사용 중이면:

```bash
pygbag --port 8080 main.py
```

### 빌드 옵션

```bash
# 빌드만 하고 서버 실행 안 함
pygbag --build main.py

# 캐시 없이 빌드
pygbag --no-cache main.py

# 디버그 모드
pygbag --debug main.py
```

## ❓ 문제 해결

### "포트가 이미 사용 중입니다" 오류

다른 포트 사용:
```bash
pygbag --port 8080 main.py
```

### "모듈을 찾을 수 없습니다" 오류

패키지 재설치:
```bash
pip install --upgrade arcade pygbag
```

### 모바일에서 접속이 안 됨

1. **PC와 모바일이 같은 Wi-Fi에 연결되어 있는지 확인**
2. **PC 방화벽 설정 확인** (포트 8000 허용)
3. **PC의 IP 주소가 올바른지 확인**
4. **브라우저에서 `http://` 대신 `http://` 사용 (https 아님)**

### 게임이 느리게 실행됨

- 모바일: 성능 제한으로 인해 느릴 수 있음
- PC: 브라우저 하드웨어 가속 활성화 확인
- 네트워크: Wi-Fi 연결 상태 확인

## 📝 참고사항

- **첫 로딩 시간**: WebAssembly 변환으로 인해 첫 로딩 시 시간이 걸릴 수 있습니다
- **브라우저 호환성**: 최신 브라우저 권장 (Chrome, Firefox, Safari, Edge)
- **모바일 성능**: 모바일에서는 PC보다 성능이 낮을 수 있습니다
- **터치 컨트롤**: 현재는 키보드 입력만 지원됩니다. 모바일 터치 컨트롤은 추가 개발 필요
